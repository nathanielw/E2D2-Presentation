[Intro] ...and I've been working on the Expeditee Desktop.

Expeditee Is...

  • Hypermedia environment for organising ideas
  • Unique UX
  • Visually outdated

So Expeditee is a hypermedia enviroment for creating and organising various types of content.

Its primary use would be for things such as note-taking, but it aims to support a wide range of activities.

It provides quite a unique style of interaction, but at the same time feels quite outdated visually, so switching between it and other desktop programs can sometimes be a little jarring.

Expeditee Desktop

  • Modernize Expeditee
  • Expand it to the entire desktop

The aim of this project was to expand Expeditee so that its core features could be used across an entire modern desktop environment.

...and the platform that was chosen for this was Android.

Hacking Android

Directly

  • Direct access to Android source
  • Long build times
  • Deployed as custom ROM
  • Hard to maintain
  • User must have root access
  • Install wipes device data
  • Can't use with custom ROMs

The possibility of modify and compiling up a custom version of Android was breifly explored, and while this was surprisingly straight forward, it would obviously not be the best approach for end-users.

Hacking Android

With Xposed

  • Mods hooks into method calls
  • Can only inject code before & after methods, not inside
  • Deployed as Android apk
  • Easy to maintain
  • User must have root access
  • Installed like normal app
  • Automatic updates
  • Works with most Android versions (4.0.3+)

Instead I switched over to creating a standard Android app that, provided the user has root access, makes the necessary changes, without permanently modifying any of the Android system. This was achieved through a really cool framework called Xposed, which handles all the hard work of injecting the changes at boot.

The end result of this was that across the Android system, the text content of elements that extend Android’s standard Text or Button UI elements text can be picked up with a right click and carried between apps, with text entry fields allowing the same mouse button combos you’d use in Expeditee for manipulating text.

The second part of the project consisted of porting the main Expeditee app to Android.

The Expeditee codebase has no real separation between graphics and logic code, so I had to make some significate changes to its structure...

The code that the Android version is running on has been structured so that the user interface is completely separate from the logic, which means, in theory, the desktop version could be modified to be running of a common codebase.

At the moment only basic viewing and navigation is supported in the app, so there's still a lot that could be added in the future, including editing, integration with the first part of the project and potentially other Android-specific features.
