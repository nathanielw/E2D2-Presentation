[Intro] ...and I've been working on 'E2D2', or 'the Expeditee Desktop'.

Expeditee Is...

  • Hypermedia environment for organising ideas
  • Unique UX
  • Visually outdated

So Expeditee is a hypermedia enviroment for creating and organising various types of content.

Its primary use would be for things such as note-taking, but it aims to support a wide range of activities beyond that.

It provides quite a unique style of interaction, but at the same time feels quite outdated visually, so switching between it and other desktop programs can sometimes be a little jarring.

Expeditee Desktop

  • Modernize Expeditee
  • Expand it to the entire desktop

The aim of this project was to expand Expeditee so that its core features work across an entire modern desktop environment.

...and the platform that was chosen for this was Android.

Hacking Android

Directly

  • Direct access to Android source
  • Long build times
  • Deployed as custom ROM
  • Hard to maintain
  • User must have root access
  • Install wipes device data
  • Can't use with custom ROMs

I breifly explored the possibility of directly modifing the Android source to add Expeditee-like features, and while this was surprisingly straight forward, it would obviously not be the best approach for end-users.

Hacking Android

With Xposed

  • Mods hooks into method calls
  • Can only inject code before & after methods, not inside
  • Deployed as Android apk
  • Easy to maintain
  • User must have root access
  • Installed like normal app
  • Automatic updates
  • Works with most Android versions (4.0.3+)

So I quickly made the switch to a really cool framework called Xposed,
which allows you to inject modifications into any Java code that the Android system executes. This is all done at runtime, so there's no need for users to install a custom Android distro. It's also easier to maintain as a developer, since the code base you're dealing with is significantly smaller.

Result

  • Right-click to pick up text*, drop into text fields
    *anything that's a descendant of android.view.TextView

The end result of this is that across the Android system, text content can be picked up with a right click and carried between apps, and then dropped into places such as text entry fields.

Result

  • Hover to focus, caret follows cursor
  • Custom cursor indicates possible actions

The content of text entry fields can also be manipulated in the same way, in addition to other Expeditee-style features, such as the text caret following the cursor. I also modified the pointer itself to give an indication of which button combos can be used at any particular time.

Porting Expeditee

The second part of my work consisted of porting the main Expeditee app across to Android.

Porting Expeditee

Desktop Expeditee:

  • Graphics mixed in with logic
  • Direct port would require also porting java.awt

Android Expeditee:

  • Graphics code completely separate from logic
  • Future ports should only require implementing graphics interfaces

Unfortunately, the current Expeditee codebase has no real separation between graphics and logic code, which makes it quite a challenge to port between system.

So for the Android version, I focused on rewriting Expeditee's core logic to be platform-agnostic, with the user interface code kept completely separate. While I only had time to port a small portion of Expeditee, the structure means that, in theory, a more complete version of the Android core could be ported back to desktop so that they share a common codebase.

Result: Basic Expeditee Viewer

This is the Expeditee app in action.
